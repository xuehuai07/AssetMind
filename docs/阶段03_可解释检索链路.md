# 阶段 03：可解释检索链路

## 目标

实现基础但可解释的知识资产检索能力，为 Agent 问答提供可追踪上下文。

## 开发内容

- 新增 `SearchResult` 类型。
- 实现 query normalization 和轻量 tokenization。
- 按标题、标签、正文进行加权评分。
- 返回 top 3 检索结果，包含 `snippet`、`score`、`matchedTerms`。
- 新增 `POST /api/search`。

## 阶段结果

- 搜索 “AIOS 支持哪些能力” 能命中 “AIOS 平台介绍”。
- 搜索 “权限 控制 可观测性” 能命中 “Agent 工作流”。
- 搜索 “客户案例 产品说明” 能命中 “数字资产知识库”。
- 无命中时返回空数组，不伪造结果。

## 验证

- 代表性 query 的 top 结果符合预期。
- 检索结果包含可解释的分数、片段和命中词。
- `npm run lint` 和 `npm run build` 通过。

## 风险记录

- 当前是关键词检索，不是语义检索。
- 中文分词采用轻量策略，召回能力有限。
