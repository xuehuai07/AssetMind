# 阶段 05：工作台 UI 与交互

## 目标

把 API 和业务能力组织成一个可操作的 RAG 工作台，突出资料、检索、问答、引用和 Agent Trace。

## 开发内容

- 实现资料列表、新增资料、问题输入、答案展示、引用来源、检索结果和 Trace 面板。
- 覆盖 loading、empty、error、no result、新增成功等状态。
- 支持提交问题后展示回答、引用和检索结果。
- 做响应式适配，桌面三栏、窄屏纵向堆叠。

## 阶段结果

- 首屏是可操作工作台，不是营销页。
- 新增资产成功后 UI 立即更新。
- 提问后展示答案、引用和 Trace。
- 检索结果和引用能说明答案依据。

## 后续升级

本阶段初版 UI 后续已根据中国用户使用习惯升级为更清爽的 Gemini 风格首页：管理功能收进工具按钮，首页优先展示居中的提问入口，证据检索改为回答后的可展开选项。

## 验证

- `npm run lint`、`npm run build`、`git diff --check` 通过。
- 使用测试 JSON 文件验证新增资料、搜索、提问和无命中场景。

## 风险记录

- Trace 和检索信息默认展示会干扰主任务，因此后续已改为用户主动展开。
