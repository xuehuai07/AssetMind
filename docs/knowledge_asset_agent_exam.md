# 笔试

> 5月28日修改

## 背景

我们正在构建企业级 AI Agent 与数字资产知识库平台。这次希望考察候选人的全栈能力、前端审美、AI 产品理解、检索链路意识和工程质量。

我们更看重产品判断、工程结构、代码质量和交互体验，而不是功能堆砌。

---

## 一、目标

实现一个简单的 Web App：知识资产问答工作台。

候选人需要完成以下能力：

1. 展示知识资产列表；
2. 支持新增知识资产；
3. 支持对知识资产进行检索；
4. 支持用户向 Agent 提问；
5. Agent 根据检索结果返回答案；
6. 展示回答和引用来源；
7. 展示简单的检索过程或 Agent Trace；
8. 页面需要有清晰、克制、有质感的 UI。

---

## 二、推荐技术栈

推荐使用以下技术栈，但不强制：

- Next.js / React
- TypeScript
- Tailwind CSS
- shadcn/ui 可选
- Node.js API Route / Express / NestJS 均可
- SQLite / JSON File / Memory Store / localStorage 均可

允许使用 AI Coding 工具，但最终代码需要候选人能够解释清楚。

---

## 三、功能要求

### 1. 知识资产列表

页面需要展示知识资产列表，每条知识资产包含：

```ts
type KnowledgeAsset = {
  id: string;
  title: string;
  content: string;
  tags: string[];
  createdAt: string;
}
```

请内置至少 3 条初始数据：

```text
1. AIOS 平台介绍
AIOS 是一个面向企业的智能体操作平台，支持知识库、工具调用、工作流编排和多智能体协作。

2. 数字资产知识库
数字资产知识库用于沉淀企业文档、业务流程、销售资料、客户案例和产品说明，并支持智能检索和问答。

3. Agent 工作流
Agent 可以通过任务拆解、工具调用、上下文记忆和结果校验完成复杂任务，但需要可观测性和权限控制来保证可靠性。
```

---

### 2. 新增知识资产

提供一个表单，支持新增知识资产。

字段包括：

- Title
- Content
- Tags

提交后，新增资产应出现在知识资产列表中。

数据可以存在内存、localStorage、JSON 文件或 SQLite 中。不限实现方式，但需要在 README 中说明取舍。

---

### 3. 简单检索能力

实现一个基础检索逻辑。

不强制要求真正接入向量数据库。可以采用以下任一方案：

- 关键词匹配
- 简单分词 + 相似度评分
- 本地 embedding / 第三方 embedding API
- 模拟向量检索，但需要保留清晰的数据结构

检索需要返回 top 3 相关知识片段，并在 UI 中展示匹配结果。

返回结构建议：

```ts
type SearchResult = {
  assetId: string;
  title: string;
  snippet: string;
  score: number;
}
```

---

### 4. Agent 问答

用户输入问题后，系统需要完成以下流程：

```text
用户问题
  ↓
检索相关知识资产
  ↓
基于检索结果生成回答
  ↓
展示答案 + 引用来源
```

不强制接入真实 LLM。可以使用 mock answer，但必须体现“基于检索结果回答”的逻辑。

例如：

用户问题：

```text
AIOS 支持哪些能力？
```

返回答案：

```text
AIOS 支持知识库、工具调用、工作流编排和多智能体协作等能力。
```

引用来源：

```text
引用 1：AIOS 平台介绍
引用 2：Agent 工作流
```

如果接入真实 LLM，可以加分，但不是必须。

---

### 5. Agent Trace / 检索过程

请展示一个简单的 “Agent Trace” 或 “检索过程” 区域，包含：

- Query：用户问题
- Retrieved Assets：命中的知识资产
- Scores：相似度分数
- Final Answer：最终答案

这个部分可以非常简单，但要能看出候选人理解 AI 系统不是黑盒。

---

### 6. UI 要求

页面需要具备基本产品完成度：

- 布局清晰；
- 层级分明；
- 组件命名合理；
- 交互状态完整；
- Loading / Empty / Error 状态至少覆盖一部分；
- 视觉不要粗糙堆砌；
- 风格干净、克制、可用。

不要求设计很复杂，但希望有“产品感”和“工程审美”。
